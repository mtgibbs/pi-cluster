# Runbook — launch the review-hub test-coverage loop

**What this is:** an unattended `ralph-qwen` loop that hands the local Q8 coder **9 tasks** — one
unit-test file per review-hub validator — each mirroring the worked example
(`scripts/reviewhub/validators/test_dependency_update.py`, 24 passing tests). Every task is gated by
`verify.sh`; the loop commits each on pass and **stops for a human if a task can't pass after 3 tries.**

Files in this dir: `spec.md` (the contract) · `verify.sh` (the deterministic gate) · `tasks.txt` (the 9 tasks).

---

## 1. Prereqs — check these first

```bash
# (a) OFF VPN. oc must reach the Beelink (.70), not a VPN resolver.
#     The old "unable to verify the first certificate" error = a VPN DNS hijack.

# (b) Beelink in WORK mode (Q8, not the 30B):
ssh beelink-ai 'cat /opt/ai-stack/.aimode_state'      # expect: work
ssh beelink-ai 'sudo aimode work'                     # if not — wait ~1 min for the Q8 to load

# (c) oc is wired (key from Keychain, reaches LiteLLM):
oc run "reply with: ok"                               # should print ok, no key/cert error
```

## 2. Get the launch-ready worktree

```bash
cd ~/dev/pi-cluster && git fetch origin && git worktree prune
# If /tmp/oc-rhtests still exists, skip this. Otherwise recreate it from the pushed branch:
git worktree add /tmp/oc-rhtests reviewhub-test-coverage \
  || git worktree add -b reviewhub-test-coverage /tmp/oc-rhtests origin/reviewhub-test-coverage
```

> The branch already holds the seed: `conftest.py`, `verify.sh`, `spec.md`, `tasks.txt`, and the
> `test_dependency_update.py` worked example. Sanity-check the gate before you start:
> `cd /tmp/oc-rhtests && bash specs/reviewhub-test-coverage/verify.sh` → `VERIFY OK`.

## 3. Start it

```bash
cd /tmp/oc-rhtests
OC_RUN_TIMEOUT=480 scripts/ralph-qwen.sh specs/reviewhub-test-coverage
```

Runs ~9 tasks (~1–1.5 hrs). Leave it — it commits each verified validator-test to the branch. Run it
in a terminal you can glance at, or background it and watch the commits.

## 4. Monitor

```bash
git -C /tmp/oc-rhtests log --oneline        # one commit accrues per validator
```

- **Loop stops with `✋ STOP … needs a human` (exit 2):** that validator's source differs from the
  pattern (different verdicts/globs/aggregation). Open its `.py` + the half-written `test_*.py`, fix the
  assertions, re-run — or delete that line from `tasks.txt` to skip it and continue the rest.

## 5. When it finishes → PR

```bash
cd /tmp/oc-rhtests
git log --oneline                                       # review the commits
git push -u origin reviewhub-test-coverage
gh pr create --base main \
  --title "test(reviewhub): validator unit-test coverage" \
  --body "Unit tests for all 10 validators' pure surface (build_prompt/parse/aggregate/_render_pr/applies_files). Generated by the ralph-qwen loop; seed + worked example by Claude."
```

**Review the diff yourself** — the Q8 is a literal stamper, so confirm the assertions are real (not
`assert True`) and match each validator's actual values. Then merge, and clean up:

```bash
git -C ~/dev/pi-cluster worktree remove --force /tmp/oc-rhtests
```

## Why this is safe to run unattended

Worktree-isolated (can't touch your checkout) · commits to a **branch only**, never pushes/merges (nothing
near the cluster) · **verify-gated** so a skipped/trivial file fails (`<10` tests / wrong import / `<4`
functions) · 480s watchdog (no hangs) · **stops for a human** when stuck. Worst case by end of day: a
branch of mediocre tests you reject.
