# Recap — Homepage retro-theme + widget health (2026-06-11 → 06-12)

## What we set out to do
Restyle the lab homepage (`home.lab.mtgibbs.dev`) with a 1970s-sci-fi terminal look and
consolidate its wasteful layout. That surfaced several broken widgets, which we then chased to
root cause.

## Shipped

### 1. "NOSTROMO terminal" theme (`specs/homepage-retro-theme/`)
- SDD spec + deterministic `verify.sh` authored first; widget-capability audit (gethomepage
  **v1.4.5**, source-tag verified) recorded in spec §13b.
- `configmap.yaml`: new `custom.css` key (initContainer copies it to `/app/config/`), plus
  `settings.yaml` theme/layout keys. Iterated over four rounds (tuning log §14):
  - **R1** green phosphor → **R2** muted **teal + darkened-orange** errors (complementary grade),
    icons monochromed into "terminal-pane" badges (grayscale→sepia→hue-rotate filter) →
    **R3** themed the widget **API-error banners** (`bg-rose`→`--error-orange`) → **R4** killed a
    **washout**: homepage's `color: teal` paints a bright page bg; fixed with a solid-dark card
    surface + a `BG:FORCE` `[class*="bg-theme-"]` override + readable mid-teal fonts.
- Layout consolidated **11 groups → 6 decks** (COMMAND / COMMS / AI CORE / REC DECK /
  ACQUISITION / CARGO BAY); dropped 4 redundant node tiles + the label-only Unbound tile.
  The mechanical regroup was generated by **qwen3-coder** one-shot (tools-off text gen) — clean
  pass, zero misses.
- Gate refactor: token checks are **presence-based** (not fixed-hex) so taste tweaks don't fight
  `verify.sh`.

### 2. Synology → QNAP cleanup

### 3. Widget diagnostics (the widgets were honestly reporting real problems)
- **Tailscale** — widget API token expired (401). Minted a new key (user), updated
  `op://pi-cluster/tailscale/api-key`, refreshed ESO + rolled. ✅ Fixed. (Recurs ≤90 days.)
- **Pi-hole secondary** — 401. Root cause: the widget used a **primary-only app-password**
  (`pihole/api-key`); app-passwords are per-instance and the secondary never had one. No 2FA, so
  app-passwords were never 2FA-gated. Generated a dedicated app-password on the secondary via its
  API (`GET /api/auth/app` → `PATCH webserver.api.app_pwhash`), stored it at
  `op://pi-cluster/pihole/secondary-api-key`, pointed the secondary widget there. ✅ Fixed.
- **Loki** — *(initially misread as a false alarm)* the `customapi` hit
  `/loki/api/v1/index/stats` with **no LogQL query**, so Loki 400'd (`parse error:
  unexpected $end`). Silent in homepage's logs (a 400 isn't a connection error), loud on
  Loki's side. Fixed by adding `query={source="heroku"}` (the Vector log-drain label; Loki
  defaults the window to 1h). ✅
- **Log Drain** (`logs.mtgibbs.dev`) — false-red dot: it's a **POST-only** Heroku log-drain,
  so a GET health probe returns 405. Dropped the `siteMonitor` → plain link. ✅
- **LazyLibrarian** — cosmetic `http→https` siteMonitor redirect (dot only). Left as-is.

> Diagnostic note: homepage's pod logs only record *connection-level* widget failures
> (ECONNREFUSED, redirect errors). An upstream **4xx** (Loki 400, Log Drain 405) renders the
> card's error state but logs nothing — so "not in the homepage logs" ≠ "healthy." Check the
> upstream's own logs (Loki did record the 400).

### 4. Docs
- `docs/flux-gitops.md`: **reconcile the git source before the Kustomization** — a Kustomization
  reconcile applies whatever revision the source currently holds, so reconciling only the
  Kustomization after a push can silently re-apply the *old* revision. Plus: verify the served
  artifact actually changed (byte-size), not just "pod Ready".

## Lessons
- **Widgets are honest narrators** — three "errors" were a dead backend (Immich), an expired key
  (Tailscale), and a per-instance-credential gap (Pi-hole). Diagnose the server path; don't
  blame the dashboard.
- **"Too bright" and "washed out" were the same bug** (low contrast) pulling the font lever
  opposite ways — the real fix was the background/surface.
- **`op` from automation has a short biometric window** — re-`op signin` then run the op step
  immediately, batched into one call.
